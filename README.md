## 1. Модификации программы

В ходе выполнения лабораторной работы программа была расширена и модифицирована для поддержки параллельных вычислений.

### 1.1. Автоматический запуск серии тестов  

Программа выполняет вычисления для набора размеров матриц:

200, 400, 800, 1200, 1600, 2000.

Также выполняются эксперименты с разным количеством потоков:

1, 2, 4, 8.

Результаты автоматически сохраняются в файл `timings_omp.txt`.

---

### 1.2. Параллелизация с использованием OpenMP  

Основные вычислительные циклы были распараллелены с помощью директив OpenMP:

- инициализация матриц (`#pragma omp parallel for`)
- умножение матриц (`#pragma omp parallel for collapse(2)`)

Это позволило задействовать несколько вычислительных ядер.

---

### 1.3. Раздельное сохранение данных  

Для каждого значения N формируются файлы:

- `A_N.txt` — первая матрица  
- `B_N.txt` — вторая матрица  
- `C_N.txt` — результат  

Сохранение выполняется только для одного потока (1), чтобы избежать лишних затрат.

---

### 1.4. Проверка корректности  

Используется Python-скрипт с NumPy:

- вычисляется эталонное решение  
- сравниваются результаты  
- оцениваются ошибки

## 2. Выполнение работы

### 2.1. Реализация алгоритма

Реализован классический алгоритм умножения матриц:

O(N³)

Для параллельной версии используется OpenMP.

---

### 2.2. Проведение экспериментов

Для каждого сочетания:

- размера матрицы N  
- количества потоков p  

выполняются:

1. Генерация матриц  
2. Умножение  
3. Замер времени  
4. Сохранение результата  

---

### 2.3. Метрики

Используются следующие показатели:

#### Ускорение (Speedup)
\[
S = \frac{T_1}{T_p}
\]

#### Эффективность
\[
E = \frac{S}{p}
\]

- Для малых размеров влияние распараллеливания незначительно  
- Наблюдается эффект насыщения (diminishing returns)
